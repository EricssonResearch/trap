TrapHostingTransport

implements TrapTransport
com.ericsson.research.trap.spi.TrapHostingTransport

Class Overview

Defines interfaces for transports that are capable of hosting files. This allows an application to sideload information on transports, instead of having to set up its own hosting setup. The primary use-case for hosting transports is when the main application wants to provide some file/data (e.g. setup JavaScript) without having to provide its own hosting interface. HTTP transports should implement this interface.

This interface is NOT intended as a replacement to the Servlet API or similar containers, and should only be used when it is too expensive to set up a proper container. Additionally, it is not intended for dynamic content, or anything advanced really.

public abstract URI addHostedObject (TrapHostingTransport.TrapHostable hosted, String preferredName)

Adds an object to be hosted. The hosted object will be accessible from the URI returned. The URI type is transport dependent; an HTTP transport would return an HTTP URI, other transports may have other types.

The URI returned will be pseudo-random. It is impossible to guarantee any specific URI, format or path, as all depend on the specific transport quantities.

The hosted object will be weakly referenced. To remove the object, simply let it get GC'd.

Parameters
hosted The object to host.
preferredName A human readable name at which this object would prefer to be hosted. The path is not guaranteed, but it may be more predictable than purely random. May be null, in which case a random path is used.
Returns
  • The URI at which the object's data can be accessed remotely.
