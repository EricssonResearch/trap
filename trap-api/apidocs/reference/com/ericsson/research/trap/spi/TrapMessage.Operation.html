Class Overview

This subclass defines the possible message operations of a TrapMessage. For Trap, this is the complete subset of operations supported, for this version, and may not be extended without requiring a clean break. That is, Trap endpoints should never silently ignore invalid operations, so remote Trap endpoints MUST ensure they do not send commands that are invalid.

public static final TrapMessage.Operation ACK

Used by transports to acknowledge receipt of a message. Transports that fail sending messages MUST trigger ttMessagesFailedSending. The ACK operation is reserved for transport usage for that purpose.

public static final TrapMessage.Operation CHALLENGE

Sent to send some authentication challenge data.

public static final TrapMessage.Operation CLOSE

CLOSE is sent from both sides to acknowledge a closing transport.

public static final TrapMessage.Operation END

END is sent to terminate a TrapEndpoint connection, including all transports.

public static final TrapMessage.Operation ERROR

Denotes an error has occurred. The connection should be terminated.

public static final TrapMessage.Operation FRAGMENT_END

Marks the end of a fragmented message. If the channel is in regular mode, this is when the reconstructed message should be dispatched.

public static final TrapMessage.Operation FRAGMENT_START

Marks the beginning of a fragmented message. The data of this message, including all MESSAGE objects that follow, and terminated by (and including the data of) the FRAGMENT_END message are actually a single message, and should be represented as such to the application, unless streaming mode has been enabled in setStreamingMode(boolean).
