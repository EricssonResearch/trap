public interface

TrapAuthentication

com.ericsson.research.trap.auth.TrapAuthentication

Class Overview

Provides an interface by which applications can authenticate Trap sessions and messages. Trap ships with no built-in implementation of TrapAuthentication; it must be entirely user-supplied. Authentication can be set per-endpoint using setAuthentication(TrapAuthentication), or per-transport using setAuthentication(TrapAuthentication).

When set, the authentication instance will be called by the transport or endpoint whenever an unauthenticated incoming message is received, and before any potentially unauthenticated message is sent. Essentially, if the remote party can trust a message (e.g. because it's transmitted over a stateful TLS connection), this authentication will not be invoked. On the other hand, if an unsecured transport is used (e.g. plain HTTP), authentication will be invoked for every message. Most transports will lie somewhere in-between, most likely invoking authentication on session start.

The basic steps when Trap receives a message is to do the following:

  • Parse the incoming message, separating auth payload from data payload. Verify the message type.
  • If message, do
    • Ask the assigned TrapAuthentication instance to verify the authentication. If successful, return.
    • If unsuccessful, ask the assigned TrapAuthentication instance to createAuthenticationChallenge(TrapMessage, Map).
    • Send the new challenge to the remote side..
  • If challenge, do
    • Ask TrapAuthentication to {@link #createAuthenticationResponse(TrapMessage, TrapMessage, Map, Map).
    • Send the challenge response back to the other side.

public abstract Collection<String> getContextKeys (Collection<String> availableKeys)

Fetches a collection of keys (strings) that this TrapAuthentication instance wants from the TrapTransport. The TrapAuthentication instance must not change this collection after this call, and must not assume the transport will call this function more times than one. It can assume the transport calls this function at least once.

As an argument, the transport supplies the context keys available from this transport. The returned collection may not contain a key that does not exist in the availableKeys collection. If the TrapAuthentication's implementation requires a context value whose key not exist in availableKeys, it may generate a value, as long as it does not significantly compromise the integrity of the authentication.

If there is not enough context information for this TrapAuthentication instance to successfully work, it may throw a TrapException.

Parameters
availableKeys A collection containing all keys that the TrapTransport can fill in with meaningful values.
Returns
  • A collection of keys that the TrapAuthentication instance wants the TrapTransport to provide on every call.
Throws
TrapException If there is not enough context information for this TrapAuthentication instance to successfully work

public abstract boolean verifyAuthentication (TrapMessage message, Map<String, String> headers, Map<String, Object> context)

Verifies the authentication of a message. Implementations should check that getAuthData() is a valid authentication string/data. Implementations must treat all arguments as read-only, and may not assume any specific ordering on the received messages.

Parameters
message The message for which to verify the authentication for. The contents of the message MUST NOT be modified.
headers A map (String, String) of eventual other transport headers. The presence and/or absence of these headers is dependant on the transport; for example, socket will have zero headers whereas HTTP may have plenty.. May contain any number of headers (including zero). May not be null. May not be modified by verifyAuthentication.
context A non-null map of the context values requested by this TrapAuthentication in getContextKeys(Collection). Every key that was returned by getContextKeys MUST be filled in.
Returns
  • true if the authentication string is correct, false otherwise (incorrect, could not be verified, etc).
Throws
TrapAuthenticationException If the authentication failure is so severe that the transport should be disconnected. This exception can be thrown to signify that the transport is behaving improperly (potential DDoS/hack) and should be terminated.
