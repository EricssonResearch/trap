Public Methods

public static TrapClient createClient (String configuration, boolean autoConfigure)

Creates a client endpoint instance, capable of creating an outgoing connection. Unlike the listener object, the client must be configured with proper configuration before it can connect. The listener endpoint instance is capable of creating a complete configuration profile out of the box, which can just be supplied to the client.

Parameters
configuration A configuration representing, at the very least, one transport address to connect to a Listener Trap Endpoint.
autoConfigure Enables/disables automatic client configuration. When true, the client can learn of additional transports and configuration parameters from the server. false makes the client ignore updated configuration.
Returns
  • A TrapEndpoint capable of opening an outgoing connection to a Listener Trap Endpoint.
Throws
TrapException If an error occurred during instantiation; either if the configuration was invalid, or if the Trap implementation was not present.

public static TrapListener createListener (String configuration)

Creates a listener endpoint instance, capable of accepting new incoming connections. The listener will be initialised with some default values, but will not listen for incoming connections until listen(OnAccept) is called.

Parameters
configuration The configuration string to use when creating the listener. It is strongly recommended you always have a configuration string, even though Trap can supply default values.
Returns
  • A newly created TrapListener instance.
Throws
TrapException If an error occurred during instantiation; either if the configuration was invalid, or if the Trap implementation was not present.

public static TrapListener createListener ()

Creates a listener endpoint with default configuration. It is up to the caller to ensure proper configuration of the listener before calling listen(OnAccept) .

Returns
  • A newly created TrapListener instance.
Throws
TrapException If an error occurred during instantiation; either if the configuration was invalid, or if the Trap implementation was not present.

public static String getVersion ()

Accessor for the current Trap API version. It is possible, though not recommended to use a different version of trap-api and trap-core. When using Maven, the versions should be matched automatically. This method has two primary purposes:

First, a trap-core implementation should verify that it matches this API version.

Second, someone with an unknown trap-full.jar can use this method to determine the version number.

Returns
  • The Trap API version, or "unknown" if not known. The version corresponds to the Maven version, so can be, e.g., 1.0 or 1.1-SNAPSHOT.
