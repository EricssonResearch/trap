Class Overview

A common ancestor interface for TrapEndpoint and TrapListener. Methods common to TrapEndpoints and TrapListeners will be found here, generally on managing the transports themselves.

Public Methods

public abstract void configure (String configuration)

Configures this TrapEndpoint, overwriting any previous configuration, and setting the new string to the new configuration. This will also reconfigure any constituent transports. This method should be used before using any of the programmatic configuration methods, as it may override them.

Parameters
configuration A string representing the new configuration.

public abstract void configureTransport (String transportName, String configurationKey, String configurationValue)

Changes the configuration of a given transport. This is an alias to accessing the transport and configuring it directly. After configuration, the transport settings will be updated.

Care should be taken when using this method to not conflict with Trap's general management. In most cases, an endpoint will manage transport settings automatically, although some tweaks can be done on a per-transport basis.

General configuration keys can be found as static properties in TrapTransport. Specific options are relevant only to the given transport.

Parameters
transportName The name of the transport to configure.
configurationKey The unprefixed configuration key
configurationValue The new value of the key.
Throws
TrapException If the transport does not exist (and thus cannot be configured).

public abstract void disableAllTransports ()

Convenience method to disable all transports at a given configurable object. This is primarily used to create controlled-environment Trap connections, where the available transports are not automatically set. Primarily targeted towards tests or other controlled environments, or when all transports need to be clamped down.

public abstract void disableTransport (String transportName)

Disables a transport. Unlike enableTransport(String), this method will always succeed, even if the transport does not exist. This allows applications to safely remove transports they do not wish to use – if the transport does not exist, it will not be used.

Parameters
transportName The transport to disable

public abstract void enableTransport (String transportName)

Enables a transport with a given name. Enabling a transport merely switches the enabled flag of the transport in the resident Trap configuration. It does not necessarily connect it.

Enabling a transport after a client endpoint has been asked to connect will cause the transport to be ignored until the reconnect procedure is triggered. For this reason, it is recommended that enabling/disabling transports is done before an endpoint is connected.

Parameters
transportName The transport to enable.
Throws
TrapException If the transport does not exist. This exception is thrown to prevent applications that rely on a certain transport from being surprised when the transport never connects. A typical case is a client disabling the http transport when websockets is not available.

public abstract String getConfiguration ()

Gets the configuration of this endpoint. This is the configuration as it applies for this node; it does not represent the configuration another endpoint needs to have to connect here. See getClientConfiguration() for that. This method is useful for debugging.

Returns
  • A string representing the current configuration of this TrapEndpoint.

public abstract TrapHostingTransport getHostingTransport (String protocol)

Attempts to fetch a TrapHostingTransport instance for the specific protocol. This method effectively fetches the hosting transports and attempts to find the one with the highest priority that conforms to protocol. Not all protocols may have hosting transports. No verification will be performed that the query can ever return a non-null value.

Parameters
protocol The protocol of the transport that should host. See TrapTransportProtocol.
Returns
  • The hosting transport for the given protocol with the highest priority. Else, null.
See Also
  • getHostingTransports()

public abstract Collection<TrapHostingTransport> getHostingTransports ()

Ask the endpoint to make a list of the currently available TrapHostingTransports associated with it. The number of transports may be zero when called from, for example, client endpoints. See the TrapHostingTransport documentation for discussion on the uses of these.

Returns
  • A collection of all transports that can potentially host objects. The collection may be empty.

public abstract TrapTransport getTransport (String transportName)

Accesses a single transport by name. This is useful for advanced configuration of transports, debugging or highly specialised tweaking.

Parameters
transportName The name of the transport.
Returns
  • The TrapTransport instance representing the transport
Throws
TrapException If the transport does not exist.

public abstract Collection<TrapTransport> getTransports ()

Gets the current set of transports associated with this TrapEndpoint. These transports represent all the instances available to the endpoint, not necessarily the ones that are currently in use. Each transport has an individual state that determines if it is connected or not.

Returns
  • A collection of transports associated with this endpoint.

public abstract boolean isTransportEnabled (String transportName)

Queries if a transport is enabled. Preferable to calling getTransport(String) and isEnabled() as this method will not throw if a transport does not exist.

Parameters
transportName The transport whose state to query.
Returns
  • true if a transport exists, and is enabled, false otherwise.

public abstract void setDelegate (TrapDelegate delegate, boolean replaceAllExisting)

Sets the object delegate as the listener for all the implemented TrapDelegate sub-interfaces. If replaceAllExisting is true, then all callbacks are reset, even the ones not implemented by delegate (they will instead log a warning if applicable). Otherwise, only the implemented callbacks are overwritten.

Multiple invocations to setDelegate can be used to set different object as delegates for different actions. For example, setDelegate(openDelegate, false) and setDelegate(dataDelegate, false) will retain both objects as delegates. The actions a delegate is retained for depend on the interfaces implemented.

All delegates can be cleared by invoking setDelegate(null, true).

Parameters
delegate The delegate to add. It must implement at least one sub-interface of TrapDelegate, or the call will have no effect.
replaceAllExisting true to clear all delegates before assigning this delegate, false to keep any delegates that do not conflict with this one.

public abstract void setDelegateContext (Object delegateContext)

Trap delegates may have an additional context object provided at callback time. This is a single object shared between all delegates, and may be used for server dispatching purposes.

Parameters
delegateContext The context object to provide to delegates.
